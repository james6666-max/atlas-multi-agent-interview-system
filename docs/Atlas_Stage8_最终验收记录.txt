五、已知问题

1. PowerShell 中文输出可能乱码，这是终端编码问题，不代表系统错误。
2. OCR 可能把部分英文字符识别错，例如 AI -> Al，需要演示时选择清晰字体和较大字号。
3. Whisper 首次加载 faster-whisper 模型较慢，首次语音测试需要等待。
4. 前端部分历史中文文案存在乱码，但功能链路和接口数据正常。后续若要正式展示中文 UI，建议单独做一次 UI 文案编码清理。

六、后续优化建议

1. 清理前端历史乱码文案，统一中文 UI。
2. 为 OCR 增加更强的后处理纠错，例如 AI / API / OCR 常见误识别。
3. 为 Whisper 增加录音时长、音量提示和超时提示。
4. 后续如需更强 RAG，可再引入轻量向量检索，但当前 Stage 8 不建议继续加复杂度。
5. 如果演示机器性能较弱，可将 ATLAS_WHISPER_MODEL 设置为 tiny。
